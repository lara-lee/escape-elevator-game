1. 개요

플레이어는 퇴근을 위해 엘리베이터를 출발시키려 하지만, 끊임없이 탑승하려는 사람들 때문에 방해받는다. 스트레스를 관리하며 최대한 많은 층을 이동(= 높은 점수 획득)하는 것이 목표다.

  • 승리 조건: 없음 (엔드리스). 최고 점수 갱신이 목표.
  • 패배 조건: 스트레스 게이지 100% 도달.
  • 핵심 루프: 문닫기 → 출발 판정 → (성공: 층수↑·점수↑·스트레스↓ / 실패: 문 재개방·스트레스↑) → 반복.

2. 용어 정의

용어 정의
층수 (floor) 누적 출발 성공 횟수. 시작값 1.
점수 (score) 출발 성공 시 누적되는 점수.
스트레스 (stress) 0~100 범위의 게이지. 100 도달 시 게임 오버.
출발 판정 문이 일정 시간(3초) 방해받지 않으면 출발 성공.
승객 (passenger) 랜덤 생성되어 엘리베이터로 접근하는 방해 요소.
콤보 연속 출발 성공 시 적용되는 보너스(확장 기능).

4. 상세 기능 명세

F-001 게임 시작

항목 내용
설명 게임을 초기 상태로 시작한다.
트리거/입력 메인 화면 시작 버튼 클릭(pointerdown)
선행 조건 메인 화면 진입 상태
처리 floor = 1, score = 0, stress = 0, door = "open", gameOver = false, 승객 목록 초기화, 게임 루프 시작
출력 게임 화면으로 전환, HUD(층수/점수/스트레스) 표시
예외 없음

F-002 문닫기

항목 내용
설명 플레이어가 문닫기 버튼을 눌러 문 닫힘을 시도한다.
트리거/입력 문닫기 버튼 클릭(pointerdown)
선행 조건 door === "open", gameOver === false
처리 문 닫힘 애니메이션 시작, door = "closing", 출발 판정 타이머 시작
출력 문 상태 시각 변경
예외 이미 닫힘/닫는 중이면 입력 무시

F-003 출발 판정

항목 내용
설명 문이 일정 시간 동안 방해받지 않으면 출발 성공으로 판정한다.
조건 문 닫힘 상태 3초 유지(door === "closing" 동안 승객 충돌 없음)
처리(성공) floor += 1, score += 100, stress -= 10(최소 0), 승객 목록 초기화, door = "open"
처리(실패) F-006(승객 충돌)으로 분기
출력 다음 층 이동 연출
예외 판정 도중 게임 오버 시 즉시 중단

F-004 승객 생성

항목 내용
설명 랜덤한 승객이 일정 주기로 생성된다.
트리거 생성 타이머 (1~5초 랜덤 간격)
처리 확률 표(6장)에 따라 승객 타입 결정, 화면 우측에 생성하여 승객 목록에 추가
출력 승객 등장
예외 gameOver === true 이면 생성 중단

F-005 승객 이동

항목 내용
설명 승객이 엘리베이터 방향으로 이동한다.
처리 매 프레임 승객을 타입별 속도(delta time 적용)로 좌측 이동, 엘리베이터 도착 여부 판정
출력 승객 위치 갱신
예외 화면 밖 이탈 승객은 목록에서 제거

F-006 승객 충돌

항목 내용
설명 승객이 문이 닫히기 전에 엘리베이터에 도착한다.
트리거 승객 위치가 엘리베이터 영역 도달
처리 문 재개방(door = "open"), 타입별 스트레스 증가, 출발 판정 타이머 취소
출력 출발 실패, 스트레스 게이지 상승
예외 충돌로 stress >= 100 도달 시 F-010(게임 오버)로 분기

F-007 스트레스 시스템

항목 내용
설명 플레이어의 스트레스 수치를 관리한다.
범위 최소 0, 최대 100
증가 규칙 일반 사원 +5 / 인턴 +3 / 택배 기사 +10 / 부장님 +20
감소 규칙 출발 성공 -10
출력 스트레스 게이지 UI 갱신
예외 0 미만으로 내려가지 않음, 100 도달 시 게임 오버 트리거

F-008 점수 시스템

항목 내용
설명 출발 성공 시 점수를 획득한다.
계산식 score += 100
추가 보너스 연속 성공 시 콤보 점수 적용 가능(F-013, 선택)
출력 점수 UI 갱신
예외 없음

F-009 층수 시스템

항목 내용
설명 엘리베이터가 출발할 때마다 층수가 증가한다.
계산식 currentFloor += 1
출력 층수 UI 갱신
예외 없음

F-010 게임 오버

항목 내용
설명 스트레스가 한계에 도달하면 게임을 종료한다.
조건 stress >= 100
처리 게임 루프 정지, gameOver = true, 승객 생성 중단, 최고 점수 비교·저장(F-012)
출력 결과 화면 표시(최종 점수, 도달 층수, 통계)
예외 없음

F-011 다시하기

항목 내용
설명 결과 화면에서 게임을 재시작한다.
트리거/입력 다시하기 버튼 클릭(pointerdown)
처리 F-001(게임 시작)과 동일하게 상태 초기화 후 게임 화면 진입
출력 새 게임 시작
예외 없음

F-012 최고 점수 저장 (권장)

항목 내용
설명 역대 최고 점수를 저장·표시한다.
처리 게임 오버 시 score와 저장된 bestScore 비교, 갱신 시 LocalStorage에 저장
저장소 LocalStorage (storage.js 추상화 계층 경유)
출력 메인/결과 화면에 최고 점수 표시
예외 LocalStorage 미지원/차단 시 메모리 값으로 폴백

F-013 콤보 보너스 (선택)

항목 내용
설명 연속 출발 성공 시 추가 점수를 부여한다.
계산식 score += 100 * comboMultiplier (연속 성공마다 배수 증가)
처리 출발 실패 시 콤보 초기화
출력 콤보 UI 표시
예외 없음

6. 데이터 모델

GameState

{
  score: 0,         // 점수
  floor: 1,         // 층수
  stress: 0,        // 스트레스 (0~100)
  door: "open",     // "open" | "closing"
  passengers: [],   // Passenger[]
  combo: 0,         // 연속 성공 횟수 (F-013)
  bestScore: 0,     // 최고 점수 (F-012)
  gameOver: false

Passenger

{
  type: "employee",  // "employee" | "intern" | "courier" | "manager"
  speed: 3,          // 이동 속도
  stressValue: 5,    // 충돌 시 스트레스 증가량
  x: 0               // 현재 x 좌표
}

7. 상태 전이

현재 상태 이벤트 다음 상태
Ready (메인) 시작 버튼 (F-001) Playing
Playing (문 열림) 문닫기 버튼 (F-002) Door Closing
Door Closing 3초 무방해 (F-003 성공) Playing (층수+1)
Door Closing 승객 충돌 (F-006) Playing (문 열림, 스트레스+)
Playing stress >= 100 (F-010) Game Over
Game Over 다시하기 (F-011) Playing

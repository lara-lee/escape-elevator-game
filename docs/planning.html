1. 프로젝트 개요

프로젝트명

퇴근 엘리베이터 (Escape Elevator)

장르

캐주얼 웹게임 / 아케이드 / 코미디

플랫폼

  • Web Browser
  • PC 우선
  • 모바일 대응 가능

목표

퇴근을 위해 엘리베이터를 출발시키려 하지만 끊임없이 탑승하려는 사람들 때문에 방해받는다. 플레이어는 스트레스를 관리하며 최대한 많은 층을 이동해야 한다.

2. 게임 컨셉

한 줄 소개

"오늘도 퇴근에 실패했습니다." 문이 닫히기 직전마다 사람들이 뛰어오는 현실을 게임으로 구현한 캐주얼 웹게임.

핵심 재미

  • 반복적인 도전
  • 랜덤 이벤트
  • 현실 공감
  • 점수 경쟁
  • 짧은 플레이 타임

3. 게임 목표

승리 조건

없음 최대한 높은 점수를 획득하는 것이 목표

패배 조건

스트레스 게이지가 100% 도달

4. 게임 흐름

Start ↓ 엘리베이터 대기 ↓ 문닫기 버튼 클릭 ↓ 문 닫힘 시도 ↓ 사람 등장 여부 판정 ↓ 성공 → 층수 증가 실패 → 문 재개방 ↓ 스트레스 증가 ↓ 반복 ↓ 스트레스 100% ↓ 게임 오버

11. 개발 단계 & PWA 전환 계획

핵심 전제: PWA는 웹앱을 다시 만드는 것이 아니라 "웹 + 얇은 레이어(manifest + service worker)" 다. 따라서 1차(웹)를 PWA를 염두에 두고 만들면, 2차(앱)는 재작업이 아니라 추가 작업만으로 끝난다.

11.1 단계 구성

단계 목표 산출물
1차 (Web) MVP 게임 완성 + PWA 준비 항목 반영 HTTPS로 배포된 플레이 가능한 웹게임
2차 (PWA) 설치형 앱 + 오프라인 플레이 + 모바일 UX 홈 화면 설치 가능한 PWA
3차 (선택) 앱스토어 출시 TWA(Android) / Capacitor 래핑

11.2 1차(웹)에서 미리 지켜야 할 설계 원칙

PWA 전환 비용을 0에 가깝게 만드는 핵심 준비 항목.

  • 모바일 우선 + 터치 입력: click 대신 pointerdown(터치+마우스 통합) 사용. 핵심 조작이 "문닫기 버튼"이므로 필수.
  • 로직과 화면 분리: state(score / floor / stress / passengers) ↔ 렌더링(DOM 갱신)을 분리해 이식성·테스트성 확보.
  • 저장소 추상화: storage.js 한 곳에서만 LocalStorage 접근 → 추후 IndexedDB 교체 시 한 파일만 수정.
  • 에셋·경로 로컬화: 외부 CDN·절대경로 금지(상대경로 사용) → 서비스워커 오프라인 캐싱 보장.
  • 게임 루프: requestAnimationFrame + delta time 기반 → 기기별 속도 균일.
  • 뷰포트 고정: viewport meta로 확대·스크롤 차단, 100dvh 사용(모바일 주소창 대응).

11.3 2차(PWA)에서 추가하는 것

  • manifest.webmanifest: name, short_name, display: standalone, orientation: portrait, theme_color, 아이콘 세트(192/512)
  • Service Worker: cache-first 전략 → 오프라인 플레이 지원
  • 앱 아이콘 + 스플래시 스크린
  • beforeinstallprompt 후킹 → "홈 화면에 추가" 설치 유도 UI
  • 앱 감성 폴리시(선택): 진동(Vibration API, 충돌 시), 화면 꺼짐 방지(Wake Lock), 풀스크린

11.4 기술 결정

  • 렌더링: Canvas 대신 DOM + CSS 채택. 이 게임 규모·캐주얼 성격·짧은 개발기간엔 DOM이 더 빠르고 PWA/반응형과 잘 맞는다.
  • 배포: HTTPS 필수(PWA 요구사항). GitHub Pages / Vercel / Netlify 모두 기본 HTTPS 제공.
